## 🌱 What we're about

Software, like everything else, decays — dependencies rot, assumptions expire, yesterday's "best practice" becomes today's footgun. **Perish Dev** leans into that instead of fighting it: we treat code as something that lives, ages, and is continually composted into better ideas.

We build small, focused, well-documented projects and let the weak parts perish so the strong ones can evolve.

## 🤝 Contributing

We welcome contributions of every size. Before you start, please read:

- 📜 [Contributing Guide](https://github.com/perishdev/.github/blob/main/CONTRIBUTING.md) — workflow, branch policy, and naming conventions
- 💚 [Code of Conduct](https://github.com/perishdev/.github/blob/main/CODE_OF_CONDUCT.md)
- 🤖 [AI Policy](https://github.com/perishdev/.github/blob/main/AI_POLICY.md)

> Open your pull requests against **`main`** from a branch named per the [Conventional Branch](https://conventional-branch.github.io/) spec.
